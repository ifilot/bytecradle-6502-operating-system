# BCOS

BCOS is a basic operating system for the ByteCradle 6502. The OS offers simple
I/O routines for keyboard handling and terminal output. It also has the drivers
to interface with the SD-card.

## Jump table ABI

BCOS jump-table artifacts are generated from a single source-of-truth table in:

- `src/os/bcos/generate_bcos_artifacts.py`

Generated outputs:

- `src/os/bcos/jumptable.inc` (generated; not tracked)
- `src/os/bcos/jumptable.s` (generated; not tracked)
- `src/os/bcos/sbc.cfg` (generated; not tracked)
- `programs/.generated/io.h`
- `programs/monitor/jumptable.inc` and `programs/monitor/jumptable.s` (generated by monitor-specific script; not tracked)

Current ABI layout:

- `BCOS_JT_BASE` starts at `$FFDC`.
- `$FFE5..$FFF7` keep the stable public API used by existing programs.
- `$FFDC..$FFE2` are reserved ABI/expansion slots:
  - ABI major (`BCOS_JT_ABI_MAJOR`)
  - ABI minor (`BCOS_JT_ABI_MINOR`)
  - one reserved slot for future extension
- The assembler asserts that the jump table ends exactly at vector start
  (`$FFFA`) to prevent silent overlap.

Regenerate artifacts manually:

```bash
./src/os/bcos/generate_bcos_artifacts.py
```

Or use Make integration:

```bash
make -C src/os/bcos generate
make -C programs generate
```

The monitor uses its own generated shim from `programs/monitor/generate_monitor_jumptable.py`,
which reads `src/os/bcos/jumptable.inc` and emits monitor-local jump-table files.


## Compilation

Ensure you have [cc65](https://cc65.github.io/) installed on your system. Simply
run `make` to compile everything. The resulting `sbc.rom` needs to be flashed
to the first bank of the SST39SF040 EEPROM.
